# 달빛선생 프리미엄 명리 기준서 노선 QA 체크리스트

## 1. 라우팅 체크

### 핵심 진입 라우트

- [ ] `/`
- [ ] `/saju/new`
- [ ] `/sample-report`
- [ ] `/membership`
- [ ] `/dialogue`
- [ ] `/about-engine`

### 결과 화면 라우트

- [ ] `/saju/[slug]`
- [ ] `/saju/[slug]/overview`
- [ ] `/saju/[slug]/premium`

### 확인 항목

- [ ] 각 라우트가 200 응답 또는 정상 렌더링된다
- [ ] 헤더/로그인/멤버십/마이페이지 동선이 끊기지 않는다
- [ ] 이전 라우트에서 새 CTA로 이동했을 때 의도한 페이지가 열린다
- [ ] 존재하지 않는 전용 상품 라우트는 fallback 경로로 안전하게 연결된다

## 2. CTA 체크

### 홈 / 샘플 / 결과 / 멤버십

- [ ] `내 명리 기준서 만들기`
- [ ] `샘플 리포트 보기`
- [ ] `엔진 기준서 보기`
- [ ] `심층 리포트로 이어보기`
- [ ] `PDF로 소장하기`
- [ ] `달빛선생에게 이어서 묻기`

### 확인 항목

- [ ] 링크 클릭 시 이동 경로가 올바르다
- [ ] 버튼/링크 스타일이 primary / secondary / muted 위계에 맞다
- [ ] disabled 상태인 CTA는 클릭 오해를 주지 않는다
- [ ] 준비 중인 CTA는 준비 중임이 명확히 보인다

---

## 3. 모바일 체크

### 뷰포트

- [ ] `360px`
- [ ] `390px`
- [ ] `768px`
- [ ] `1024px 이상`

### 확인 항목

- [ ] 홈 Hero CTA 2개가 세로로 자연스럽게 쌓인다
- [ ] 긴 제목이 잘리지 않고 2~3줄 안에서 읽힌다
- [ ] 카드 그리드가 어색하게 한 줄에 눌리지 않는다
- [ ] 판정 근거 패널이 화면 밖으로 넘치지 않는다
- [ ] 멤버십 카드와 소장 가치 카드의 버튼이 겹치지 않는다
- [ ] 결과 화면 상단 `1분 요약 + 판정 근거 + 소장 가치` 흐름이 모바일에서도 무너지지 않는다

---

## 4. 카피 체크

- [ ] 적중률 단정 표현이 없다
- [ ] 건강 단정 표현이 없다
- [ ] 투자 단정 표현이 없다
- [ ] 법률 단정 표현이 없다
- [ ] 공포성 길흉 표현이 없다
- [ ] `AI가 추측하지 않는다` 메시지가 과도하게 반복되지 않는다
- [ ] `소장`, `기준`, `근거`, `대화` 메시지가 서로 다른 맥락으로 자연스럽게 배치된다

### 금지 표현 예시 재확인

- [ ] `반드시 망한다`
- [ ] `대흉이니 피해야 한다`
- [ ] `질병이 생긴다`
- [ ] `치료된다`
- [ ] `이 종목을 사라`
- [ ] `소송에서 이긴다`

---

## 5. 데이터 체크

### 빈 값 / 제한 입력 / 미구현 상태

- [ ] `decisionTrace` 없음
- [ ] `metadata` 없음
- [ ] `time unknown`
- [ ] `PDF 기능 없음`
- [ ] `로그인 안 됨`
- [ ] `프로필 없음`

### 확인 항목

- [ ] `decisionTrace`가 없어도 fallback 패널로 화면이 깨지지 않는다
- [ ] `metadata`가 없어도 판정 근거 패널이 자연스럽게 보인다
- [ ] 태어난 시간이 없을 때 보수적 해석 안내가 뜬다
- [ ] PDF가 준비 중이어도 CTA/카드 문구가 어색하지 않다
- [ ] 비로그인 상태에서 결과보관함/대화/결제 진입이 치명적으로 깨지지 않는다
- [ ] 프로필이 없어도 홈 개인화/결과/멤버십 진입이 안전하게 fallback 된다

## 최종 QA 메모

- [ ] 홈 → 샘플 리포트 → 결과 → 멤버십 → 대화 흐름이 하나의 브랜드 경험으로 이어진다

- [ ] 기존 계산 엔진/판정 로직을 건드리지 않고도 UI/카피/전환 구조가 분리되어 있다
